## Research

### What a terminal theme actually needs to cover

Pi uses 24-bit RGB escape codes for all its own output (detected via COLORTERM=truecolor), so the terminal palette (16 ANSI colours) doesn't affect pi's rendering at all. However, the terminal palette DOES affect:
1. The naked terminal background — visible between pi rendering regions and around the input box
2. Shell prompts (oh-my-bash uses palette indices like \e[32m, \e[34m)
3. System tools: ls, git diff, man pages, etc.
4. Kitty's own chrome: tab bar, cursor, selection highlight

The critical alignment is: Kitty's `background` colour must match alpharius `bg` (#06080e) so the void surrounding pi's rendered blocks is seamless. Kitty's `foreground` should match alpharius `fg` (#c4d8e4). Without this, the "warm brass on void" contrast relationship that makes #b89020 read as brass-gold instead of plain yellow breaks.

For the 16 ANSI colour slots: map alpharius signal colours to the standard semantic roles (red→error, green→success, yellow→warning/numbers, blue→accent, etc.). This makes oh-my-bash and ls output visually coherent with the alpharius vocabulary even though pi doesn't use these slots.

### Kitty theme format and ANSI slot mapping

Kitty themes are `.conf` snippets (can live at ~/.config/kitty/current-theme.conf, included from kitty.conf). Key fields:
  background, foreground, cursor, selection_background, selection_foreground
  color0–color15 (the 16 ANSI slots: 0-7 normal, 8-15 bright)

Standard semantic mapping for ANSI slots:
  color0=black (bg-adjacent), color8=bright-black (dim text)
  color1=red (error), color9=bright-red
  color2=green (success), color10=bright-green
  color3=yellow (warning), color11=bright-yellow
  color4=blue (links/accent), color12=bright-blue
  color5=magenta (special), color13=bright-magenta
  color6=cyan (info), color14=bright-cyan
  color7=white (fg-adjacent), color15=bright-white

Alpharius mapping:
  color0=#06080e (void bg), color8=#344858 (dimFg)
  color1=#c83030 (red/error), color9=#e04040 (brighter red)
  color2=#1ab878 (green/success), color10=#20d890
  color3=#b89020 (yellow/warning), color11=#d4aa30
  color4=#1a8898 (primaryMuted), color12=#2ab4c8 (primary/accent)
  color5=#6060c0 (ai/purple), color13=#8080d0
  color6=#4aa8c0 (syntaxType/cyan), color14=#6ecad8 (primaryBright)
  color7=#607888 (mutedFg), color15=#c4d8e4 (fg)

cursor=#2ab4c8 (accent, the ceramite teal)
selection_background=#102030 (selectedBg)
selection_foreground=#c4d8e4 (fg)

### Where to put the output and how to keep it in sync

Options:
A) Static file checked into repo at themes/alpharius.conf (Kitty) — simple, manual sync when palette changes
B) Generated by a script/extension from alpharius.json — stays in sync automatically
C) Extension command `/theme:export kitty` that writes to ~/.config/kitty/current-theme.conf — active install

Option A is the right starting point: the palette rarely changes, a static conf file is easily auditable, and users can include it directly. Option C (active install command) is a nice addition on top. A script that regenerates from the JSON is worth having so a palette change doesn't require manual updating of the conf.

The extensions/style.ts file already reads/applies the theme — adding an export function there is natural.

For discoverability: README.md should mention the Kitty theme and link to the conf file. The style SKILL.md should reference it as the "ground truth alignment" step.

## Decisions

### Decision: Static conf file + generator script, no active-install extension command

**Status:** decided
**Rationale:** The Kitty theme is a setup-time concern, not a runtime concern. A static `themes/alpharius.conf` checked into the repo is the right artifact: auditable, simple to use (`include ~/.../alpharius.conf`), and usable without running any extension command. A small generator script (`scripts/export-kitty-theme.ts`) regenerates it from alpharius.json when the palette changes, ensuring they stay in sync. The `/theme:export` command idea is deferred — it adds UX complexity for a one-time setup action. Start with static + generator.

## Implementation Notes

### File Scope

- `themes/alpharius.conf` (new) — Kitty terminal theme: background/foreground/cursor/selection + color0-15 mapped from alpharius.json palette
- `scripts/export-kitty-theme.ts` (new) — Generator script: reads themes/alpharius.json, writes themes/alpharius.conf. Run manually when palette changes.
- `skills/style/SKILL.md` (modified) — Add terminal alignment section: reference alpharius.conf, explain Kitty include directive

### Constraints

- themes/alpharius.conf must be valid Kitty conf syntax (key = value, comments with #)
- color0-15 must follow standard ANSI semantic roles so oh-my-bash and ls output align with alpharius vocabulary
- cursor colour must be accent (#2ab4c8) — matches pi's own focus/interactive colour
- selection_background must match theme selectedBg (#102030)
- Script must derive all values from alpharius.json, not hardcode them
