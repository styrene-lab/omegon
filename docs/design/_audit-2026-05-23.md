# docs/design stale-content audit — 2026-05-23

## Scope

Initial audit of `docs/design/` for old, out-of-date, or OBE content. The directory currently contains 46 markdown design nodes.

This audit starts with inventory and triage. It does not assert that every stale reference should be deleted: many files are useful as historical decision journals. The cleanup question is whether each file is still current reference material, historical-but-useful, superseded, or removable.

## Repository reality checks

Current implementation has moved substantially from the older TypeScript extension architecture described by many design docs:

- The only top-level `extensions/` source currently present is `extensions/omegon-browser/`; paths such as `extensions/project-memory/index.ts`, `extensions/model-budget.ts`, `extensions/effort/index.ts`, `extensions/dashboard/footer.ts`, and `extensions/bootstrap/index.ts` are absent.
- Core runtime features now live in Rust under `core/crates/omegon/src/features/`, including `model_budget.rs`, `memory.rs`, `cleave.rs`, `lifecycle.rs`, `delegate.rs`, `usage.rs`, and related TUI modules.
- The native TUI has a Rust dashboard/statusline implementation under `core/crates/omegon/src/tui/`, so older docs about `pi-tui` extension overlays and TypeScript dashboard widgets are no longer current implementation references.
- `flake.nix`, `.github`, and `Justfile` do not currently show Cachix configuration evidence (`cachix`, `nixConfig`, `extra-substituters`, or `CACHIX_AUTH_TOKEN`).

## Proposed cleanup taxonomy

Use these labels consistently during the next pass:

- **current-reference** — doc reflects current Rust implementation and can be used as guidance.
- **historical-decision** — doc is a useful journal/ADR but should not be used as current file-scope reference.
- **superseded** — durable point has been replaced by a newer design; add a pointer to the replacement and avoid duplicate guidance.
- **remove/archive** — no remaining unique value after useful facts are moved elsewhere.
