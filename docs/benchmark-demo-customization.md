# Demo customization framework — teams fork and adapt for their codebase

## Overview

The omegon-demo repo becomes a template. Teams fork it, replace the Rust project with their own codebase, edit the phase prompts to match their workflows (e.g. Phase 3 writes TypeScript instead of Rust, Phase 6 creates domain-specific design nodes). The framework structure (9-phase walk-through, results schema, /demo launcher) stays the same — the content is customizable. Needs: documentation on how to customize, a 'make your own demo' guide, and validation that the results schema works across different project types.
