# Lifecycle hygiene — verification substates and binding normalization

## Intent

Harden the OpenSpec/design-tree lifecycle so task-complete changes cannot linger in ambiguous verifying state. Expose concrete verification substates such as missing assessment, stale assessment, missing binding, reopened work, and archive-ready; normalize binding truth so OpenSpec status, archive gating, and design-tree lifecycle metadata all agree on whether a change is bound.
