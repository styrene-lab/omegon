# Post-Assess Reconciliation — Update OpenSpec and Design Tree after Review/Fix Cycles

## Intent

Extend lifecycle reconciliation beyond post-cleave and pre-archive into the review/fix loop. After `/assess spec`, `/assess cleave`, or follow-up fixes, the harness should reconcile remaining work: refresh tasks.md partial completion honestly, append design-tree implementation notes when scope changed, and keep dashboard state aligned with what review actually discovered.
